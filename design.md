# AI Omni-Drive 모바일 인터페이스 설계

## 설계 목표

AI Omni-Drive는 **세로형 9:16 화면**과 **한 손 조작**을 기준으로 설계된 운전 보조 인터페이스입니다. 전체 경험은 iOS 기본 앱처럼 단순하고 즉시 읽히는 위계를 유지해야 하며, 주행 중에는 장식보다 **신호 상태, 속도, 진행 방향, 경고 상태**가 가장 먼저 눈에 들어와야 합니다. 새로 추가되는 기능은 전방 카메라가 감지한 **날씨·환경 조건**에 따라 적색 점멸 경고의 환경 프리셋을 자동 전환하여, 사용자가 설정 화면에 들어가지 않아도 현재 시야 조건에 맞는 경고 강도를 받을 수 있게 하는 것입니다.

## 화면 목록

| 화면 | 주요 목적 | 핵심 사용자 가치 |
|---|---|---|
| 홈 HUD | 현재 주행 상태와 신호 상태를 가장 크게 보여주는 메인 화면 | 한눈에 신호, 속도, 방향, 경고 상태를 확인 |
| 카메라 인식 화면 | 전방 카메라로 신호등과 주변 환경을 실시간 분석 | 실제 인식 결과와 스캔 상태를 즉시 확인 |
| 설정 화면 | 음성, 경고, 스캔, 환경 프리셋을 조정 | 사용자 시력·주행 습관에 맞춘 개인화 |
| 홈 화면 전문 설정 센터 | HUD 요소 위치·크기·폰트·테마 조정 | 개인별 시인성과 배치 최적화 |

## 화면별 콘텐츠와 기능

### 1. 홈 HUD

홈 HUD는 **상단 대형 신호 카드**, **중단 주행 정보 카드**, **하단 방향 및 제어 바**로 구성합니다. 상단 카드에는 STOP, SLOW, GO와 같이 즉시 이해 가능한 텍스트와 신호 상태 색을 크게 표시합니다. 중단 영역에는 현재 속도와 핵심 보조 정보만 남겨 시각적 부담을 줄입니다. 하단에는 방향 화살표와 카메라·홈·설정 진입 버튼을 배치해 엄지손가락 하나로 누를 수 있도록 합니다.

새 자동 환경 프리셋 기능은 홈 HUD에서 **현재 적용 중인 환경 프리셋**, **자동 전환 여부**, **카메라가 읽은 환경 요약**이 짧고 분명한 문장으로 표시되어야 합니다. 사용자는 현재 프리셋이 수동인지 자동인지, 자동이라면 왜 전환되었는지를 바로 이해할 수 있어야 합니다.

### 2. 카메라 인식 화면

카메라 인식 화면은 상단에 라이브 상태 배지와 스캔 상태를 두고, 중앙에 카메라 프리뷰와 인식 프레임을 배치합니다. 하단에는 인식 범위 조절, 최근 인식 결과, 보조 기능 상태, 마지막 스캔 시각이 배치됩니다. 이 화면은 **실시간 인식의 근거 화면**이므로, 날씨 기반 자동 프리셋 전환 기능도 가장 명확하게 노출해야 합니다.

카메라가 비·안개·야간·맑은 주간과 같은 환경을 읽으면, 이 화면에는 **감지 환경 라벨**, **자동 전환된 프리셋 이름**, **자동 적용 규칙 설명**이 함께 보여야 합니다. 예를 들어, "우천 반사 감지 · 우천 프리셋 자동 적용"과 같이 짧은 조합으로 제시합니다.

### 3. 설정 화면

설정 화면은 음성 가이드, 스캔, 진동, 저시력 모드, 적색 점멸 강도, 환경 프리셋, 우선순위 모드, 감도 모드를 카드형 섹션으로 나눕니다. 각 카드에는 현재 상태를 설명하는 짧은 문장을 넣어 사용자가 바꾸는 옵션의 영향을 이해할 수 있도록 합니다.

환경 프리셋 섹션에는 기존 수동 프리셋 선택 외에 **자동 전환 스위치**와 **자동 규칙 요약 문구**를 추가합니다. 사용자가 자동 전환을 켜면 날씨 인식 결과에 따라 프리셋이 바뀌고, 끄면 마지막 수동 프리셋이 유지됩니다. 수동 미세 조정 슬라이더를 움직이면 프리셋은 custom으로 바뀌지만, 자동 전환이 켜져 있으면 다음 유효한 환경 감지 시 다시 자동 추천 프리셋으로 덮어쓸 수 있다는 점을 분명히 안내해야 합니다.

### 4. 홈 화면 전문 설정 센터

이 화면은 HUD의 위치, 크기, 폰트, 배경 질감, 신호 강조 강도를 조정하는 전문가용 설정입니다. 자동 날씨 프리셋 기능 자체의 주 제어 화면은 아니지만, 자동 적용 후에도 홈 HUD의 글자 크기와 배치를 유지해야 하므로 시각적 일관성 검증 기준 화면으로 사용합니다.

## 핵심 사용자 플로우

| 사용자 플로우 | 단계 |
|---|---|
| 자동 전환 사용 시작 | 사용자가 설정 화면에서 자동 환경 프리셋 전환을 켠다 → 카메라 화면 또는 실시간 스캔을 시작한다 → 날씨·환경 감지 결과가 들어온다 → 추천 프리셋이 자동 적용된다 → 홈 HUD와 카메라 화면에 적용 상태가 반영된다 |
| 수동 프리셋 유지 | 사용자가 자동 전환을 끈다 → 표준/야간/우천/안개 프리셋 중 하나를 직접 선택한다 → 슬라이더 미세 조정 시 custom으로 전환된다 → 이후 자동 감지가 발생해도 현재 수동 설정을 유지한다 |
| 자동 전환 결과 확인 | 사용자가 카메라 화면에서 실시간 스캔을 본다 → 환경 감지 배지에서 "야간", "우천", "안개" 등의 결과를 본다 → 같은 시점에 홈 HUD에서도 적용 프리셋과 이유를 확인한다 |

## 자동 환경 프리셋 전환 규칙

자동 전환 규칙은 사용자 입장에서 예측 가능해야 하므로 단순하고 설명 가능하게 유지합니다.

| 감지된 환경 | 적용 프리셋 | 적용 이유 |
|---|---|---|
| night | night | 주변 광량이 낮고 전방 장면이 어두워 적색 점멸 대비를 높여야 함 |
| rain | rain | 노면 반사와 빗물 번짐으로 인해 점멸 속도와 밝기 조합을 우천 조건에 맞춤 |
| fog | fog | 대비가 낮고 윤곽이 흐려져 안개·흐림 전용 프리셋이 적합함 |
| clear 또는 unknown | standard | 특수 환경이 명확하지 않을 때는 기본 표준 주간 프리셋 유지 |

규칙 충돌 시에는 **안전 우선 순서**를 적용합니다. 즉, `fog > rain > night > standard` 순으로 우선 적용하여 가시성이 가장 낮은 조건을 먼저 반영합니다.

## 색상 선택

앱 전반은 기존 Apple-inspired HUD 스타일을 유지하되, 자동 전환 상태가 보조 정보로 명확히 드러나도록 색을 정합니다.

| 역할 | 색상 | 용도 |
|---|---|---|
| 기본 배경 | `#E7EAEE` | 연한 실버 그레이 바탕 |
| 카드 쉘 | `#F7F8FA` | HUD 카드와 패널 배경 |
| 기본 텍스트 | `#11181C` | 본문 및 상태 텍스트 |
| 보조 텍스트 | `#5B6470` | 설명 문구와 메타 정보 |
| 적색 경고 | `#C41230` | STOP 상태, 전체 점멸 경고 |
| 황색 경고 | `#FFE37A` | SLOW 상태 |
| 녹색 진행 | `#7EF36B` | GO 상태 |
| 자동 프리셋 배지 | `#DCEAFE` | 자동 적용 상태 배경 |

## 인터랙션 원칙

모든 토글과 큰 버튼은 화면 하단 또는 카드 우측 상단처럼 손이 닿기 쉬운 영역에 배치합니다. 자동 환경 프리셋 전환 스위치는 적색 점멸 경고 환경 프리셋 섹션의 가장 먼저 보이는 위치에 두어 사용자가 기능 존재를 놓치지 않게 합니다. 자동 전환이 발생했을 때는 강한 모달 대신 **짧은 상태 배지 변화와 요약 문구 갱신**으로 알려 운전 중 주의를 분산시키지 않도록 합니다.

## 구현 메모

자동 환경 프리셋 전환 기능은 설정 저장 구조, 카메라 인식 결과 저장 구조, 홈 HUD 상태 표시를 하나의 데이터 흐름으로 연결해야 합니다. 즉, 카메라 화면에서 감지한 환경 정보가 공유 저장소에 기록되고, 설정 화면은 자동 전환 스위치와 마지막 자동 감지 환경을 저장하며, 홈 HUD는 같은 정보를 읽어 현재 프리셋과 자동 적용 이유를 일관되게 보여줘야 합니다.
