# 내비게이션 연동 메모

## 카카오내비 공식 문서 확인 결과

카카오내비는 외부 앱에서 목적지를 넘겨 실행할 수 있는 URL 기반 연동 방식을 제공합니다. `navigateUrl()`과 `shareUrl()`로 카카오내비 앱 실행 URL을 만들 수 있고, 앱이 설치되어 있으면 해당 앱이 열리며 미설치 시 설치 페이지로 보내는 흐름을 권장합니다.

iOS에서는 카카오내비 앱 실행 가능 여부 확인을 위해 `LSApplicationQueriesSchemes`에 `kakaonavi-sdk`를 추가해야 합니다. 목적지는 이름과 좌표를 포함한 객체로 전달하며, 경유지도 함께 전달할 수 있습니다.

현재 프로젝트에서는 우선 Expo의 위치 권한과 실시간 위치 구독으로 현재 위치를 추적하고, 선택된 외부 내비 앱으로 목적지 연동을 여는 구조를 잡아야 합니다. 이후 홈 화면 화살표는 실제 위치 변화와 경로 포인트의 진행 방향 차이를 계산해 좌회전·우회전·직진·유턴 상태로 반영하는 방식이 적합합니다.

## T MAP 공식 문서 확인 결과

T MAP API 공식 사이트에서는 경로안내, 다중 경유지안내, 위치 조회, 교통정보 등 다양한 API와 함께 Android Navi SDK 배포 정보를 제공합니다. 즉, 티맵은 단순 화면 링크 수준이 아니라 별도의 내비 SDK 및 경로 API 기반 연동 구조를 지원하는 방향으로 보입니다.

현재 화면에서는 팝업으로 최신 Android Navi SDK 배포 공지가 노출되어 있어, 실제 앱 연동 시에는 티맵의 경로 API 또는 Navi SDK를 활용하는 구조가 적합합니다. 현재 Expo 프로젝트에서는 우선 외부 앱 선택값 저장과 URL/앱 실행 연동 구조를 먼저 만들고, 이후 필요한 API 키나 SDK 조건이 확정되면 티맵 전용 심화 연동으로 확장하는 방식이 안전합니다.
